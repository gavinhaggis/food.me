bugs & friction identified by code review

  • Bug — getSafeProducts wrong sort order: Dexie .filter().reverse().sortBy() — .reverse() is silently ignored when followed by .sortBy(). Safe foods appear oldest-first. Fix: collect with .toArray() then sort manually.
  • Bug — duplicate event listeners on scan re-entry: Every call to startScanScreen() stacks a new click and keydown listener onto the manual barcode input and torch button. After 3 navigations, one Enter keypress fires 3 API calls.
  • Friction — square bracket content not stripped by normaliseIngredient: normaliseIngredient only strips round parentheses (). Labels using [E306, E307] style pass through unnormalised. Low severity — not a matching blocker but produces messy ingredient display.
  • Friction — history list shows barcode then flickers to product name: Product names are resolved async after render. On slow devices this is visible. Fix: resolve names before rendering or show a skeleton.
  • Friction — fuzzy search requires 2+ chars: Single-character queries (e.g. typing "e" toward "egg") return nothing. Consider reducing threshold to 1 for dictionary key/displayName exact-prefix matches.
  • Friction — manual barcode "go" button on mobile: inputmode="numeric" hides the Enter key on some Android keyboards. The "go" button is present but small. Consider inputmode="tel" which shows a numeric pad with a clearer action key.
